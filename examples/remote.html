Dyno Form Example and Documentation

Simple Example

In order to generate a form, you need to ensure that you call the dynoForm with 4 properties.

form
A map with all the attribute values (name, action, method etc) values that your form will need.
fieldsets
A list of triplets, each triplet referring to a Fieldsets legend, its name and finally a map of fieldset attributes.
fields
A list of maps, each map should have a name, a label, a type. Types can be "textarea", "text", "password", "select", "checkbox" etc. In case of "select" and "checkbox", an Additional map key called "options" should be present which should be a list of pairs. each pair representing the Display name and value.
buttons
A list of maps. each map representing a button that the form will contain. each of these maps are a representation of the button attributes.
